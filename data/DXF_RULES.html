DXF 레이어 규약

arch_view 일조·태양광 시뮬레이터 — 배치도 작성 가이드

AutoCAD 등에서 배치도를 작성할 때 아래 레이어 이름·기하 규칙을 지키면 arch_view가 대지·인접대지·주동·인접건물·창면을 자동으로 구분해 불러옵니다.

1. 좌표계·단위 규칙 가장 중요

  • 단위: mm(밀리미터) 기본 — 한국 실무 mm 도면을 그대로 사용하세요. m 단위로 재저장할 필요가 없습니다.
  • 단위 자동 인식 — DXF HEADER의 $INSUNITS가 4(mm)면 ÷1000, 6(m)이면 그대로. 없으면 mm로 가정. 불러오기 화면에서 mm/m 수동 고정도 가능.
  • 정북 = 도면 +Y축 — 북쪽이 Y+가 아니면 프로그램의 "정북 보정각(northAngle)"으로 보정합니다 (도면 자체를 회전시키지 마세요).
  • 모든 레이어는 하나의 월드 좌표계(WCS)를 공유해야 합니다 — 같은 원점·같은 축척.
  • 파일을 나눠 그려도 됩니다. 단, 모두 같은 원점(0,0) 기준이어야 "추가 DXF 불러오기(병합)" 시 정확히 겹칩니다.

2. 레이어 목록

레이어명용도도형비고
PLAN_<타입>_<번호> 권장 계획 주동LWPOLYLINE (닫힘) 예: PLAN_A_1, PLAN_B_2. 타입 문자(A/B/…)는 타워형·판상형 등을 사용자가 임의로 구분. 층수는 기본 15층(프로그램에서 조정), 필요 시 PLAN_A_1_20F처럼 층수 접미사 가능. 건물명은 계획주동_A_1로 표시. 색상 7(흰색) 권장
ADJ_<타입>_<번호> 인접 건물LWPOLYLINE (닫힘) 예: ADJ_C_1 → 인접건물_C_1. 기본 5층
PLAN_BLDG(_<n>F) 계획 주동 (구버전 호환)LWPOLYLINE (닫힘) PLAN_BLDG_15F = 15층. 접미사 없으면 기본 15층
ADJ_BLDG(_<n>F) 인접 건물 (구버전 호환)LWPOLYLINE (닫힘) 접미사 없으면 기본 5층. 색상 6(자홍) 권장
SITE_BOUNDARY 대지 경계선LWPOLYLINE (닫힘) 반드시 닫힌 폴리라인. 색상 1(빨강) 권장. ADJ_BOUNDARY가 없을 때 정북사선 기준선 대체용. 내부는 3D 뷰에서 회색 바닥으로 채워져 그림자 확인용 지면이 됨
ADJ_BOUNDARY 인접대지 경계선LWPOLYLINE 열린 선 허용. 색상 2(노랑) 권장. 정북사선(정북 방향)·채광사선 (창면 직각 방향) 기준선
ROAD_CL 도로 중심선LWPOLYLINE / LINE 색상 4(하늘색) 권장. 채광사선에서 도로 중심선을 인접대지경계선으로 간주 (제86조③3호) — 같은 배율 적용
PARK_BOUNDARY 공원 등 경계LWPOLYLINE 색상 3(초록) 권장. 채광사선 기준선(도로와 동일 취급)
PLAN_WIN 계획주동 창 있는 벽면LINE / LWPOLYLINE 해당 벽면의 footprint 에지 위에 그림(벽 일부만 덮어도 됨). 색상 6(자홍) 권장
ADJ_WIN 인접건물 창 있는 벽면LINE / LWPOLYLINE 3D 하이라이트 표시 용도
CONTOUR(_<고도 m>) 등고선 (M7 지형용)LWPOLYLINE·POLYLINE·LINE 고도는 정점 Z값(또는 elevation 38) 우선, 0이면 레이어명 접미사 (CONTOUR_10 = 10m). 상세는 CONTOUR 등고선 규약 참고
  • 같은 레이어에 폴리라인이 여러 개면 각각 별도 건물로 인식되며 계획주동_A_1, 계획주동_A_1-2처럼 구분됩니다. 다만 주동 1동 = 레이어 1개를 권장합니다(이름 식별 명확).
  • 타입 문자(A/B/…)는 같은 타입 주동을 뜻하므로 한 타입 레이어의 주동은 형상이 모두 같아야(합동) 합니다. 서로 다른 형상이 섞여 있으면 불러올 때 경고합니다. 이름의 -2, -3은 도면의 폴리라인 순서로만 붙어 어느 동이 다른 형상인지 알 수 없으므로, 경고가 어긋난 동 이름을 짚어줍니다. 미러 복사본은 같은 타입으로 보아 경고하지 않습니다. 사선·인동·세대수 판정은 동별 실제 형상으로 하므로 결과에는 영향이 없습니다 — 타입이 다르면 PLAN_B_1처럼 레이어를 나누세요.
  • 경계선·도로·공원·등고선 레이어는 건물로 압출되지 않고 참고용 2D 선으로만 표시됩니다(클릭·이동 불가).

3. 각 건물 외곽선 규칙

  • 건물 1동 = 닫힌 폴리라인 1개 (LWPOLYLINE, closed flag 또는 첫점=끝점).
  • 꼭짓점 3개 이상, 자기 교차(꼬임) 없는 단순 다각형.
  • 시계/반시계 방향 무관 — 프로그램이 부호 면적으로 자동 판별.

4. 법적 검토와 레이어의 관계

정북사선 건축법 시행령 제86조 제1항 (전용·일반주거지역)

계획주동의 각 부분에서 정북 방향으로 ADJ_BOUNDARY까지의 수평거리 중 최솟값 D로 판정 (없으면 SITE_BOUNDARY 대체). 창면과 무관하며 별도 실행 버튼 없이 항상 자동 재계산 — 검토선(초록=적합, 빨강=위반)이 3D에 항상 표시됩니다.

높이 10m 이하 부분: 이격 ≥ 1.5m
높이 10m 초과 부분: 이격 ≥ 높이 × 0.5

거리 D에서의 허용높이: D < 1.5m → 건축 불가 / D ≥ 1.5m → max(10m, 2×D)

채광사선 제86조 제3항 제1호

검토 대상은 PLAN_WIN으로 표시된 계획주동 창면뿐. 창면에서 직각(법선) 방향으로 가장 가까운 기준선(ADJ_BOUNDARY / ROAD_CL / PARK_BOUNDARY)까지 수평거리 D에 대해:

높이 H ≤ 2 × D (근린상업·준주거지역은 4배로 설정 변경)

도로·공원 중심선은 제3호에 따라 인접대지경계선으로 간주 — 같은 배율. PLAN_WIN이 없는 주동은 검토 제외(방위 추정 안 함).

인동거리 제86조 제3항 제2호

두 계획주동의 벽면을 직각방향으로 확장했을 때 실제로 마주보는 쌍만 검토 (대각선으로 어긋난 배치는 검토 대상 아님).

  • 가목 — 채광창 있는 벽면이 마주봄: 이격 ≥ 0.5 × 높은 동 높이 (시행령 하한 — 서울시 조례 0.8배, 프로그램 프리셋 버튼으로 전환)
  • 라목 — 채광창 없는 벽면 ↔ 측벽: 8m 이상
  • 마목 — 측벽 ↔ 측벽: 4m 이상 (측벽 = 그 건물 최장 벽의 절반 이하 길이인 벽으로 근사 판별)

PLAN_WIN이 전혀 없는 동은 모든 벽을 "창 있음"으로 간주(보수적). 나목(남측 저층 특례)·다목(부대복리시설)은 미반영 — 필요 시 값 직접 조정.

일조권(수인한도) 대법원 판례 기준 — 법령 아님

동지일 기준 인접건물(ADJ_*) 벽면·지붕을 격자로 나눠 시간대별 태양 레이캐스팅으로 연속 2시간(9~15시) 또는 총 4시간(8~16시) 일조 확보 여부를 판정합니다. 버튼 실행 방식이며 파랑=적합, 빨강=미달.

5. 창면 그리는 법 (PLAN_WIN)

핵심: 창을 표시할 벽면의 footprint 에지(변) 위에 PLAN_WIN 레이어로 LINE(또는 LWPOLYLINE)을 그립니다. 벽면에 스냅해 그리면 되고, 허용오차 0.1m(변환 후 기준) — 벽에서 0.1m 넘게 떨어지면 인식되지 않습니다. 벽 전체를 덮지 않고 일부 구간만 그려도 됩니다(한 벽에 여러 개도 가능). 어느 쪽이든 그 벽면 전체가 "창 있는 벽"으로 판정됩니다 — 법 판정 단위가 벽면 1개이기 때문입니다.

3D 뷰에서 창이 표시된 벽면은 연한 파랑 반투명 하이라이트로 구분됩니다 (창선 길이가 아니라 벽면 전체에 표시). 좌우/상하 반전 시 창면도 함께 반전됩니다.

6. 파일 구성 예시

방법 A — 파일 1개: 위 레이어를 모두 같은 DXF에 넣고 한 번에 불러오기.

방법 B — 파일 분리:

  • site.dxf: SITE_BOUNDARY, ADJ_BOUNDARY, ROAD_CL, PARK_BOUNDARY
  • plan.dxf: PLAN_A_1, PLAN_A_2, PLAN_B_1, PLAN_WIN …
  • adjacent.dxf: ADJ_BLDG_5F(또는 ADJ_C_1 등), ADJ_WIN …

먼저 site.dxf를 "DXF 불러오기"로 불러온 뒤 나머지를 "추가 DXF 불러오기"로 병합합니다. 모든 파일이 같은 원점(0,0) 기준이어야 서로 어긋나지 않습니다.

7. 예제 파일

data/sample_site.dxf(생성 스크립트: scripts/generate_sample_dxf.js)에 위 레이어를 모두 사용한 예제가 있습니다 — mm 단위($INSUNITS=4), PLAN_A_1/PLAN_A_2/PLAN_B_1_10F 권장 규약 사용. AutoCAD에서 열어 좌표·레이어 구성을 참고하세요.
