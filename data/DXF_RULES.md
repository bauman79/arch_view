# DXF 레이어 규약 (arch_view)

AutoCAD 등에서 배치도를 작성할 때 아래 레이어 이름·기하 규칙을 지키면
arch_view가 대지·인접대지·주동·인접건물·창면을 자동으로 구분해 불러옵니다.

## 1. 좌표계·단위 규칙 (가장 중요)

- **단위: mm(밀리미터)** — 한국 AutoCAD 실무 도면은 거의 전부 mm 단위이므로,
  이 규약도 **mm를 기본값**으로 합니다. 도면을 별도로 m 단위로 재저장할 필요가
  없습니다 — 일반적으로 작업하던 mm 도면을 그대로 사용하세요.
- **프로그램이 단위를 자동 인식합니다.** DXF HEADER의 `$INSUNITS` 값을 읽어
  `4`(mm)면 좌표를 1000으로 나누고, `6`(m)이면 그대로 사용합니다.
  `$INSUNITS`가 없거나 다른 값이면 **mm로 가정**합니다(한국 실무 기본값).
  불러오기 화면의 "도면 단위"를 `mm 고정`/`m 고정`으로 수동 지정할 수도 있습니다.
- **정북 = 도면 좌표계의 +Y축**. 도면의 북쪽이 Y+가 아니면 프로그램 쪽 "정북 보정각(northAngle)"
  입력으로 보정합니다 (도면 자체를 회전시키지 마세요).
- 대지 경계선·인접대지·주동·인접건물 등 **모든 레이어는 하나의 월드 좌표계(WCS)를
  공유**해야 합니다. 즉, 같은 원점·같은 축척으로 그려야 서로 위치가 어긋나지 않습니다.
- 한 DXF 파일에 모든 레이어를 넣어도 되고, 대지/주동/인접대지를 파일별로 나눠도
  됩니다. **여러 파일로 나누는 경우 반드시 같은 원점(0,0)을 기준으로 그려서** 파일을
  각각 불러와도(사이드 패널의 "추가 DXF 불러오기") 서로 정확히 겹치도록 합니다.

## 2. 레이어 목록

| 레이어명 | 용도 | 도형 | 비고 |
|---|---|---|---|
| `PLAN_<타입>_<번호>` **(권장)** | 계획 주동 | LWPOLYLINE (닫힘) | 예: `PLAN_A_1`, `PLAN_B_2`. 타입 문자(A/B/…)는 타워형·판상형 등을 사용자가 임의로 구분하는 용도. 층수는 **기본 15층**(프로그램에서 조정), 필요하면 `PLAN_A_1_20F`처럼 층수 접미사도 가능. 건물명은 `계획주동_A_1`로 표시됨. 색상 7(흰색) 권장 |
| `ADJ_<타입>_<번호>` | 인접 건물 | LWPOLYLINE (닫힘) | 예: `ADJ_C_1`. 기본 5층. 건물명 `인접건물_C_1` |
| `PLAN_BLDG(_<n>F)` (구버전 호환) | 계획 주동 | LWPOLYLINE (닫힘) | `PLAN_BLDG_15F` = 15층. 접미사 없으면 기본 15층 |
| `ADJ_BLDG(_<n>F)` (구버전 호환) | 인접 건물 | LWPOLYLINE (닫힘) | 접미사 없으면 기본 5층. 색상 6(자홍) 권장 |
| `SITE_BOUNDARY` | 대지 경계선 | LWPOLYLINE (닫힘) | 반드시 닫힌 폴리라인. 색상 1(빨강) 권장. `ADJ_BOUNDARY`가 없을 때 정북사선 기준선 대체용으로도 쓰임. **내부는 3D 뷰에서 회색 바닥으로 채워져 그림자 확인용 지면이 됨** |
| `ADJ_BOUNDARY` | 인접대지 경계선 | LWPOLYLINE | 열린 선도 허용. 색상 2(노랑) 권장. **정북사선**(정북 방향)과 **채광사선**(창면 직각 방향) 기준선 |
| `ROAD_CL` | 도로 중심선 | LWPOLYLINE 또는 LINE | 색상 4(하늘색) 권장. 채광사선에서 도로 중심선을 인접대지경계선으로 간주(제86조③3호) — 같은 배율 적용 |
| `PARK_BOUNDARY` | 공원 등 경계 | LWPOLYLINE | 색상 3(초록) 권장. 채광사선 기준선(도로와 동일 취급) |
| `PLAN_WIN` | 계획주동 창 있는 벽면 표시 | LINE 또는 LWPOLYLINE | 해당 벽면의 주동 footprint 에지 **위에** 그림(벽 일부만 덮어도 됨). 색상 6(자홍) 권장 |
| `ADJ_WIN` | 인접건물 창 있는 벽면 표시 | LINE 또는 LWPOLYLINE | 3D 하이라이트 표시 용도 |
| `CONTOUR(_<고도 m>)` | 등고선 (M7 지형용) | LWPOLYLINE·POLYLINE·LINE | 고도는 정점 Z값(또는 elevation 38) 우선, 0이면 레이어명 접미사(`CONTOUR_10` = 10m). 상세는 `data/CONTOUR_RULES.md` 참고 |

- 같은 레이어에 주동/건물이 여러 개 있어도 됩니다 — 폴리라인마다 별도 건물로
  인식하며, `PLAN_A_1` 레이어에 폴리라인이 2개면 `계획주동_A_1`, `계획주동_A_1-2`로
  구분됩니다. 다만 **주동 1동 = 레이어 1개**로 나눠 그리는 것을 권장합니다(이름 식별 명확).
- 타입 문자(A/B/…)는 **같은 타입 주동**을 뜻하므로, 한 타입 레이어의 주동은 형상이
  모두 같아야(합동) 합니다. 서로 다른 형상이 섞여 있으면 불러올 때 경고합니다 —
  예: `PLAN_A_1 레이어에 서로 다른 형상 3종이 섞여 있습니다 (615.5㎡ 5동 / 220.3㎡ 1동
  — 계획주동_A_1-5 / …)`. 이름의 `-2`, `-3`은 **도면의 폴리라인 순서**로만 붙어 어느 동이
  다른 형상인지 알 수 없으므로, 경고가 어긋난 동 이름을 짚어줍니다.
  좌우 반전(미러) 복사본은 같은 타입으로 보아 경고하지 않습니다.
  **사선·인동·세대수 판정은 동별 실제 형상으로 하므로 결과에는 영향이 없습니다** —
  타입이 다르면 `PLAN_B_1`처럼 레이어를 나누면 목록에서 구분됩니다.
- `SITE_BOUNDARY` / `ADJ_BOUNDARY` / `ROAD_CL` / `PARK_BOUNDARY` / `CONTOUR`는
  건물 매스로 압출되지 않고 **참고용 2D 선**으로만 표시됩니다(클릭·이동 불가).

## 3. 각 건물 외곽선 규칙

- 건물 1동 = **닫힌 폴리라인 1개** (LWPOLYLINE, closed flag 또는 첫점=끝점).
- 꼭짓점은 3개 이상, 자기 교차(꼬임) 없이 단순 다각형이어야 합니다.
- 시계/반시계 방향은 상관 없습니다 — 프로그램이 부호 면적으로 자동 판별합니다.

## 4. 창면 표시(PLAN_WIN) — 채광사선·인동거리 판정에 사용

인동거리·채광사선 적용 기준은 벽면에 **창(채광창)이 있는지**에 따라 달라집니다.
`PLAN_WIN` 레이어로 계획주동의 창 있는 벽면을 표시합니다.

- **채광사선** (건축법 시행령 제86조 제3항 제1호): 검토 대상은 **`PLAN_WIN`으로
  표시된 계획주동의 벽면뿐**입니다. 창면에서 벽 **직각(법선) 방향**으로 가장 가까운
  기준선까지 수평거리 D를 재서 **높이 H ≤ 2×D**(법정 기본 — 근린상업·준주거지역은
  4배로 설정 변경)로 판정합니다. 기준선이 `ROAD_CL`(도로중심선)·`PARK_BOUNDARY`
  (공원경계)이면 제3호에 따라 그 선을 인접대지경계선으로 보아 **같은 배율**을
  적용합니다. `PLAN_WIN`이 전혀 없는 계획주동은 검토 대상에서 제외됩니다
  (방위 추정 폴백 없음 — 실제 창 위치에만 의존).
- **인동거리** (제86조 제3항 제2호): 두 계획주동의 벽면을 **직각방향으로 확장했을 때
  실제로 마주보는 쌍만** 검토합니다(대각선으로 어긋난 배치는 검토 대상 아님).
  - 가목: 마주보는 벽 중 하나라도 채광창이 있으면 이격 ≥ **0.5 × 높은 동 높이**
    (시행령 하한 — **서울시 조례는 0.8배**, 프로그램의 프리셋 버튼으로 전환)
  - 라목: 채광창 없는 벽면 ↔ 측벽 → **8m 이상**
  - 마목: 측벽 ↔ 측벽 → **4m 이상** (측벽 = 그 건물 최장 벽의 절반 이하 길이인 벽으로 근사 판별)
  - `PLAN_WIN`이 전혀 없는 동은 하위호환으로 모든 벽을 "창 있음"으로 간주(보수적).
  - 나목(남측 저층·개구부 방향 특례)·다목(부대복리시설)은 미반영 — 필요 시 값 직접 조정.
- `ADJ_WIN`은 3D 뷰의 창면 하이라이트 표시 용도로만 쓰입니다.
- 3D 뷰에서 창이 표시된 벽면은 **연한 파랑 반투명 하이라이트**로 구분됩니다.

**그리는 법**: 창을 표시할 벽면의 주동 footprint 에지(변) **위에** `PLAN_WIN` 레이어로
LINE(또는 LWPOLYLINE)을 그립니다. 벽면에 스냅해 그리면 되고, 허용오차는 0.1m
(변환 후 기준)입니다 — 벽에서 0.1m 넘게 떨어지면 인식되지 않습니다.

벽 전체를 덮지 않고 **일부 구간만 그려도 됩니다**(창 실제 위치대로 그려도 무방).
한 벽에 창선을 여러 개 그려도 되고, 어느 쪽이든 **그 벽면 전체가 "창 있는 벽"으로
판정**됩니다 — 법 판정 단위(채광사선·가/라/마목)가 벽면 1개이기 때문입니다.
따라서 3D 뷰의 파랑 하이라이트도 창선 길이가 아니라 **벽면 전체**에 표시됩니다.

## 4-1. 정북사선 — ADJ_BOUNDARY 기준 (제86조 제1항 현행)

정북사선은 창면과 무관하게, 계획주동의 **각 부분(footprint 꼭짓점·에지 중점)**에서
정북 방향(도면 Y+, `northAngle` 보정 반영)으로 `ADJ_BOUNDARY`까지의 수평거리를 재고
그중 **최솟값 D**(경계선에 가장 가까운 부분)로 판정합니다(`ADJ_BOUNDARY`가 없으면
`SITE_BOUNDARY`로 대체). 현행 기준:

- 높이 **10m 이하** 부분: 인접대지경계선에서 **1.5m 이상** 이격
- 높이 **10m 초과** 부분: 각 부분 **높이의 1/2 이상** 이격

거리 D에서의 허용높이로 환산하면 `D < 1.5m → 건축 불가`, `D ≥ 1.5m → max(10m, 2×D)`.
별도 버튼 없이 배치·정북각이 바뀔 때마다 항상 자동 재계산되며, 검토선(초록=적합,
빨강=위반)이 3D 뷰에 항상 표시됩니다.

## 5. 파일 구성 예시

**방법 A — 파일 1개**: 위 레이어를 모두 같은 DXF 파일에 넣고 한 번에 불러오기.

**방법 B — 파일 분리**: 예를 들어
- `site.dxf`: `SITE_BOUNDARY`, `ADJ_BOUNDARY`, `ROAD_CL`, `PARK_BOUNDARY`
- `plan.dxf`: `PLAN_A_1`, `PLAN_A_2`, `PLAN_B_1`, `PLAN_WIN` 등 계획 주동만
- `adjacent.dxf`: `ADJ_BLDG_5F`(또는 `ADJ_C_1` 등), `ADJ_WIN` 등 인접 건물만

먼저 `site.dxf`를 "DXF 불러오기"로 불러온 뒤, `plan.dxf`·`adjacent.dxf`를
"추가 DXF 불러오기"로 순서대로 불러오면 같은 좌표계 기준으로 병합됩니다.
**추가 불러오기는 처음 불러온 파일의 화면 중심 보정값을 그대로 이어서 쓰므로,
모든 파일이 동일한 원점(0,0) 기준으로 그려져 있어야** 서로 어긋나지 않습니다.
